# MinerU 文档解析与排版复刻 - 踩坑记录

## 一、环境配置

### 1.1 Python 版本问题
**问题**：Windows 下 MinerU 不支持 Python 3.13，`ray` 依赖会出问题
**现象**：运行 `mineru` 命令卡住不动，最终超时
**解决**：使用 Conda 创建 Python 3.12 环境
```bash
conda create -n py312 python=3.12 -y
conda activate py312
```

### 1.1.1 PATH 指向错误的 Python 环境
**问题**：系统 PATH 中 `mineru` 指向 Python 3.13 的安装，而非 py312 conda 环境
**现象**：`mineru` 命令启动后卡住，GPU 监控显示无活动（CPU 模式）
**排查**：
```python
import shutil
print(shutil.which('mineru'))  # 检查实际路径
```
**解决**：在脚本中显式使用 py312 的 mineru：
```python
PY312 = Path(r"C:\Users\29595\Miniconda3\envs\py312")
mineru_cmd = str(PY312 / "Scripts" / "mineru.exe")
```

### 1.2 PyTorch 版本问题
**问题**：`mineru[all]` 默认安装 CPU 版本的 PyTorch
**现象**：`torch.cuda.is_available()` 返回 `False`
**解决**：强制安装 CUDA 版本
```bash
pip install --force-reinstall torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128
```

### 1.3 MiKTeX 安装
**方式**：使用 winget 安装
```bash
winget install MiKTeX.MiKTeX --accept-package-agreements --accept-source-agreements
```

### 1.4 MiKTeX 字体包缺失
**问题**：Asana-Math 字体未安装
**现象**：LaTeX 编译报错 `The font "Asana-Math" cannot be found`
**解决**：
```bash
mpm --install asana-math
```

## 二、MinerU 使用

### 2.1 命令行参数
```bash
mineru -p <输入路径> -o <输出目录> -b pipeline -m ocr -s <起始页> -e <结束页>
```
- `-b pipeline`：使用 pipeline 后端（支持 CPU）
- `-m ocr`：强制 OCR 模式（扫描件推荐）
- `-s/-e`：页码范围（0-based）

### 2.2 输出文件说明
| 文件 | 说明 |
|------|------|
| `_layout.pdf` | 布局可视化 |
| `_span.pdf` | 文本片段可视化 |
| `_middle.json` | 像素级坐标数据（精确） |
| `_content_list_v2.json` | 0-1000 归一化坐标 |
| `.md` | Markdown 格式输出 |

### 2.3 坐标系统区别
**`content_list_v2.json`**：bbox 是 0-1000 归一化坐标
**`middle.json`**：bbox 是像素坐标，需要 `page_size` 字段配合使用

```python
# middle.json 坐标转换
scale_factor = layout_pdf_width / middle_page_size[0]
x_pdf = x_pixel * scale_factor
y_pdf = page_height - (y_pixel * scale_factor)  # y轴翻转
```

## 三、排版复刻踩坑

### 3.1 文本和公式混在一起
**问题**：`content_list_v2.json` 中 `paragraph_content` 混杂 `text` 和 `equation_inline`
**解决**：使用 `middle.json`，每个 span 有独立的 `bbox` 和 `type`

### 3.2 image_caption 未识别
**问题**：图片标注（"图4.10"、"(a)"等）没有显示
**原因**：存储在 `image_caption` 子块中，不在 `text` 块中
**解决**：遍历时需处理 `image.sub_type == 'image_caption'`

### 3.3 LaTeX 公式渲染慢
**问题**：每个文本都单独编译 LaTeX，37次启动太慢（>10分钟）
**解决**：文本用 reportlab 直接绘制，只有公式用 LaTeX 编译

### 3.4 Caption 偏大偏上
**问题**：image_caption 位置和大小不对
**原因**：
1. 用了 `y2`（bbox顶部）绘制，应该用 `y1`（bbox底部）
2. 字体大小等于 bbox 高度，实际应该缩小
**解决**：
```python
# 位置：用 y1（底部）
# 大小：bbox 高度 * 0.8
font_pt = h_px * scale_factor * 0.8
captions.append((content, x1, y1, font_pt))
```

### 3.5 reportlab 不能直接插入 PDF
**问题**：`c.drawImage()` 不能接受 PDF 文件
**解决**：用 pypdf 的 `merge_page()` 合并矢量 PDF

### 3.6 公式编译：standalone 多页裁剪失败
**问题**：`standalone` 类 + `minipage` + `\newpage` 批量编译时，只有最后一页能正确裁剪，前面的页面变成 A4 全尺寸
**现象**：公式 PDF 尺寸为 595×841（A4），缩放到小 bbox 后变成一个点
**解决**：使用 `preview` 包替代 `standalone`，每页独立裁剪：
```latex
\documentclass{article}
\usepackage[active,tightpage]{preview}
\begin{document}
\begin{preview}
$formula1$
\end{preview}
\newpage
\begin{preview}
$formula2$
\end{preview}
\end{document}
```

### 3.7 公式缩放：min(w/fw, h/fh) 压缩成点
**问题**：用 `min(w/fw, h/fh)` 缩放公式，bbox 高度很小但公式 PDF 高度很大，导致 scale 极小
**解决**：不能用 bbox 高度约束公式高度。应根据宽高比选择缩放策略：
- 单行公式（`fh/fw < 0.12`）：按高度缩放 `scale = th/fh`
- 多行公式（`fh/fw >= 0.12`）：按宽度缩放 `scale = tw/fw`

### 3.8 公式缩放：需同时参考 MinerU bbox 和编译 PDF 的宽高比
**问题**：仅看编译 PDF 的 `fh/fw` 会误判。LaTeX 把多行公式压成宽单行，PDF ratio 很低，被当成单行处理
**解决**：取两者的较大值 `aspect = max(pdf_aspect, bbox_aspect)`，再判断是否多行

### 3.9 interline_equation 的 content 为空
**问题**：`interline_equation` 块的 `block.content` 是空字符串，实际内容在 `block.lines[0].spans[0].content`
**解决**：
```python
content = block.get('content', '')
if not content:
    for line in block.get('lines', []):
        for span in line.get('spans', []):
            c = span.get('content', '')
            if c:
                content = c
                break
```

### 3.10 clean_latex 删除了 array 环境
**问题**：`clean_latex()` 中 `re.sub(r'\\begin\s*\{\s*array\s*\}[^}]*\}', '', s)` 把 `\begin{array}` 和列定义删掉了，导致多行公式编译成单行
**解决**：独立公式（interline_equation）用原始内容编译，不经过 clean_latex。inline_equation 才需要 clean

### 3.11 LaTeX display mode `\[...\]` 压平 array
**问题**：用 `\[...\]`（display mode）编译含 `array` 环境的公式，LaTeX 把多行 array 压成宽单行
**解决**：统一用 `$...$`（inline mode）编译所有公式，包括多行公式

### 3.12 多行公式拆行：\tag 和 \left/\right 跨行断裂
**问题**：按 `\\` 拆分 array 行时，`\tag{...}` 出现在 array 内部导致报错；`\left...\right` 跨行拆断导致大括号不匹配
**解决**：
- `\tag{...}` 提取出来，只加到最后一行
- 每行去掉 `\left` 和 `\right`（拆行后无法配对）

### 3.13 文本字体大小：同 line 内 span 应统一字号
**问题**：同一行内不同 text span 因 OCR bbox 微小差异匹配到不同 category，字号不一致
**解决**：先收集 line 内所有 text span 的字号，取最小值作为统一字号，再逐 span 做宽度校验

### 3.14 文本/公式宽度校验
**问题**：字体大小或公式缩放后内容超出 span bbox 宽度
**解决**：用 `stringWidth()` 校验文本宽度，超出则按比例缩小；公式同理检查 `draw_w > tw`

### 3.15 MiKTeX 首次编译卡住
**问题**：MiKTeX 首次编译缺少包时弹出交互式下载提示，脚本卡住
**解决**：设置环境变量 `MIKTEX_ENABLE_INSTALLER=1` 允许自动安装

### 3.16 输出文件被占用
**问题**：PDF 阅读器打开输出文件时，脚本写入报 `PermissionError`
**解决**：自动重命名，文件存在时追加 `_v2`, `_v3` 后缀

## 五、处理顺序（推荐）

1. **配置环境**：Conda + Python 3.12 + CUDA PyTorch + MiKTeX
2. **下载模型**：`mineru-models-download -s modelscope -m pipeline`
3. **解析文档**：`mineru -p <pdf> -o <output> -b pipeline -m ocr`
4. **分析 middle.json**：统计公式和文本高度
5. **编写复刻脚本**：
   - 图片：reportlab 绘制
   - 文本：reportlab 直接绘制（快速）
   - 公式：LaTeX 编译后 pypdf 合并（矢量）
   - Caption：reportlab 绘制，字体缩小 80%
6. **测试单页**：先处理一页验证效果
7. **批量处理**：确认无误后处理全书

## 六、middle.json 数据结构

### 6.1 preproc_blocks（主要内容）
| 类型 | 说明 | 子结构 |
|------|------|--------|
| `text` | 正文段落 | `lines > spans`，span type 可为 `text` 或 `inline_equation` |
| `interline_equation` | 独立公式 | content 在 `lines[0].spans[0].content`，非 `block.content` |
| `image` | 图片 | `blocks > image_body`（图片）+ `image_caption`（标注） |
| `title` | 标题 | 有 `level` 字段（1=章, 2=节），content 在 `lines > spans` |
| `chart` | 图表 | `blocks > chart_body`（图片）+ `chart_caption` |
| `table` | 表格 | `blocks > table_body`（含 html）+ `table_caption` |

### 6.2 discarded_blocks（页眉页脚）
| 类型 | 说明 |
|------|------|
| `header` | 页眉（如"第1章"） |
| `page_number` | 页码 |

### 6.3 para_blocks
与 `preproc_blocks` 类似，另含 `index` 类型（目录/索引条目）

## 七、已知限制

1. **Python 版本**：Windows 下必须用 3.12，不支持 3.13
2. **LaTeX 语法**：MinerU 输出的公式需要清理（`\frac 1 2` → `\frac{1}{2}`）
3. **中文路径**：某些情况下可能有编码问题
4. **多行公式**：LaTeX `array` 的列布局与原始排版可能不同，拆行后每行独立编译
5. **公式编译失败**：约 5% 的公式因 LaTeX 语法问题编译失败（如不支持的命令）
6. **文本处理**：当前未处理长文本段落（仅处理图片+公式+标注+标题+页眉页脚）

## 八、中文公式支持

### 8.1 MinerU 中文公式识别

**问题**：MinerU 默认不识别公式中的中文字符，输出为乱码

**解决**：设置环境变量启用实验性中文公式支持：
```powershell
$env:MINERU_FORMULA_CH_SUPPORT='true'
```

仅对 `pipeline` 后端有效。

### 8.2 LaTeX 编译中文公式

**问题**：公式中的中文在 `$...$` 数学模式下，Asana-Math 字体不包含中文字符，渲染为问号

**现象**：
```latex
$P_e = \frac{错误码元数}{传输总码元数}$
% 输出: P_e = \frac{???}{???}
```

**解决**：编译前自动将中文用 `\mbox{}` 包裹：
```latex
$P_e = \frac{\mbox{错误码元数}}{\mbox{传输总码元数}}$
% 输出: P_e = \frac{错误码元数}{传输总码元数}
```

**实现**：`latex_compiler.py` 中的 `wrap_chinese_in_text()` 函数在编译前自动处理。

## 十、输出优化

### 10.1 文件体积问题

**问题**：复刻版 PDF 体积比预期大

**原因**：
1. 图片直接嵌入原始分辨率，未压缩
2. 每个公式 PDF 独立嵌入，字体资源重复
3. reportlab 生成的页面内容未压缩

### 10.2 图片压缩

**问题**：MinerU 提取的图片是原始分辨率，但 PDF 中显示尺寸很小

**解决**：嵌入前用 PIL 压缩图片：
```python
def compress_image(img_path, max_size=1024, quality=75):
    img = Image.open(img_path)
    # 缩放到最大 1024px
    scale = min(max_size / max(img.size), 1.0)
    img = img.resize((int(img.size[0]*scale), int(img.size[1]*scale)))
    img.save(tmp.name, 'JPEG', quality=quality)
```

**效果**：大图片（1359px）压缩到 1024px，体积减少约 50%

### 10.3 qpdf 资源去重

**问题**：pypdf 合并公式 PDF 时不自动去重字体资源

**解决**：编译后调用 qpdf 压缩：
```bash
qpdf --replace-input output.pdf
```

**效果**：去除重复字体资源，体积减少约 20%

### 10.4 页面压缩

**解决**：reportlab 生成页面时启用 zlib 压缩：
```python
c.setPageCompression(1)
```

**效果**：对文本和矢量内容有效，对图片无效（已是 JPEG）

### 10.5 公式批量编译优化

**改进**：增大 `BATCH_SIZE` 到 500，减少批次数，降低字体嵌入次数

**效果**：同一 batch 内的公式共享字体资源
